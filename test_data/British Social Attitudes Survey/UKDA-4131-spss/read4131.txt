2.  Validation and conversion:

2.1 Content validation: 

    Detailed checks of data against documentation were undertaken.    
    Approximately 200 variables checked and found to be sound.  The 
    data are fully labelled and well documented. 

    For derived variables: DOMP/DOSPK/DOGOV/DOTV/DOSIGN/DORAIS/DOPROT/
                           DOGRP and DONONE

    Labels in documentation read; '1 = Yes, 2 = No'.  Labels used in the 
    data read; '0 = Wouldn't do, 1 = Would do'.  The data are correct.

    At request of the Depositor, ACORN variable removed from the data file.

2.2 Conversion process: 

    No conversion required regarding the data file as it was deposited
    as a .por file.  Documentation converted to .pdf files (portable 
    document format) from Word files via Adobe Acrobat Writer 3.0.  Two 
    .pdf files were created plus an index file to aid navigation through 
    the documentation. 

2.3 Conversion validation:

    The number of cases and variables werw checked against data/documentation 
    received.

2.4 Summary of value-added outputs: 

    Following conversion, the .pdf files were checked and bookmarked 
    thoroughly.  Variable list, frequency and descriptive files were
    produced for checking and archiving purposes.

    Value label added to variable NEWDC - 'MS = Southampton'.
